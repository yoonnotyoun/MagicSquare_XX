## 1) 주제 한 문장

> **4×4 부분 마방진을 "맞다"고 판정하기 전에, 행·열·양쪽 대각선 검산이 빠지거나 한쪽만 보고 멈추지 않게 한다.**

- 솔루션 이름(검증 프로그램·솔버·UI) 없음
- 불편(반복 검산)·비용(15~20분)·오판(한쪽 대각선만 확인)에 직접 대응

---

## 2) R-G-I-O

| | |
|---|---|
| **Role** | 4×4 부분 마방진을 손·코드로 다루는 학습자의 **검산 누락·오판**을 줄이기 위해 ECB 기반 마방진 로직을 만드는 프로그래머 |
| **Goal** | 빈 칸을 채운 판에 대해 **행 4 + 열 4 + 대각선 2**(총 10줄) 합이 34인지 확인하기 전에는 "완료"로 판정하지 않는다. 틀린 축을 **검산 단계에서** 바로 드러낸다 |
| **Input** | 4×4 정수 격자 (`grid[4][4]`), 빈 칸 2개 포함 가능, 목표 합 **34** |
| **Output** | 검산 결과: **통과** 또는 **미통과 + 어느 축(행/열/대각선·인덱스)이 34가 아닌지** |

## 3) 성공 기준 3개 (Mom Test 증거 연결)

| # | 성공 기준 | 판정 조건 | Mom Test 증거 |
|---|-----------|-----------|---------------|
| **SC-01** | **대각선 검산 누락 방지** | "맞다" 판정 전 대각선 2줄이 검산 목록에 **반드시 포함**된다. 행·열만 확인하고 종료할 수 없다 | ① *"행·열만 맞으면 된 줄 알았어"* — 대각선을 손 계산에서 빠뜨림 |
| **SC-02** | **양쪽 대각선 모두 확인** | 주대각선·반대각선 **둘 다** 합 34를 검사한다. 한쪽만 맞아도 "완료"로 처리하지 않는다 | ② *"한쪽만 맞아서 '괜찮겠지' 하고 멈췄거든"* |
| **SC-03** | **숫자 변경 후 전체 재검산** | 격자 값이 바뀌면 행·열·대각선 **전부** 다시 검산한다. 부분 확인만으로 재제출·재판정하지 않는다 | ③ *"바꾼 뒤에는 행·열·대각선 전부 다시 계산했어"* — 올바른 행동을 시스템이 기본으로 강제 |

## 4) 표면 문제 — 이번 프로젝트에서 하지 않을 것

Mom Test에서 걸러진 **솔루션 먼저** 정의. 검증 전 가설, 구현 범위 밖.

| 금지 항목 | 이유 |
|-----------|------|
| ❌ "4×4 마방진 **검증 프로그램**을 만든다" | 솔루션이 먼저. 진짜 문제는 **판정·검산 순서** |
| ❌ "대각선까지 자동 체크하는 **솔버**를 만든다" | 빈 칸 채우기·자동 해가 목표가 아님 |
| ❌ "ECB 패턴으로 **MagicSquare 앱**을 개발한다" | 앱·UI는 증거 없음 |
| ❌ "행·열·대각선 합을 한 번에 보여주는 **UI**" | 표시만으로는 한쪽 대각선만 보고 멈추는 습관을 막지 못함 |
| ❌ Mom Test 증거 없는 부가 기능 (힌트, 자동 채우기, 5×5 확장 등) | SC-01~03과 무관 |

### Rule (`.cursorrules` 또는 `.cursor/rules/`)

프로젝트 **헌법** — Agent가 항상 따를 검산·TDD 규칙.

| ID | 규칙 |
|----|------|
| R-01 | 4×4 격자, 마방진 상수 **34**. 빈 칸은 `0` 또는 별도 sentinels로 표기 (추후 PRD에서 확정) |
| R-02 | **"맞다" 판정 전** 행 4 + 열 4 + 대각선 2 = **10줄** 전부 합 34 검산 (SC-01, SC-02) |
| R-03 | 한 축이라도 34가 아니면 **완료·제출 판정 금지**. 어느 축이 틀렸는지 반환 (SC-02) |
| R-04 | 격자 값 변경 후에는 **전체 10줄 재검산** 필수 (SC-03) |
| R-05 | **Mom Test 게이트** — 아래 증거 3줄과 무관한 기능 추가 금지 |
| R-06 | TDD: RED(실패) → GREEN(최소) → REFACTOR(통과 유지). **1사이클 = 요구 1묶음** |
| R-07 | ECB: entity(격자·합)·control(검산 오케스트레이션)·boundary(입출력) 책임 분리 |

**Mom Test 게이트 (R-05) — 기능 추가 전 대조**

1. *"행·열만 맞으면 된 줄 알았어"*
2. *"한쪽만 맞아서 '괜찮겠지' 하고 멈췄거든"*
3. *"바꾼 뒤에는 행·열·대각선 전부 다시 계산했어"*

| A) RED만 — 실패 테스트 1묶음 | **Command** | — |

| C) ECB·계약 리뷰만 | Review Loop | 사이클 마감·PR 때 |

**권장 Skill (만들 경우):** `.cursor/skills/magic-square-arr-cycle/SKILL.md`

**WORKBOOK만 채울 때:** Skill 파일은 **만들지 않아도 됨**. Test Loop·Rule에 워크플로를 적어 두고, TDD 진입 시 Skill로 승격.
