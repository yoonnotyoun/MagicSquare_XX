## 페르소나

4×4 부분 마방진(빈 칸 2개)을 손 계산 또는 코드로 완성·검증하는 학습자.  
실습 맥락: 마방진 프로젝트 소개(행·열·대각선 합 34, ECB 개념 분석) 슬라이드 기반 작업.

---

## 인터뷰 답변 (과거 사실)

| 질문 | 답변 |
|------|------|
| 마지막으로 4×4 부분 마방진을 다룬 때 무엇을 했는가? | 슬라이드와 같이 작업했으나 어려움이 있었음 |
| 어디서 막혔는가? | 빈 칸 2개를 넣고 행·열·대각선 합을 맞췄으나, 대각선 하나를 빼먹어 **20분** 소모 |
| 20분 동안 무엇을 했는가? | 같은 판을 다시 검산하다가 숫자를 바꿔 넣어 보고 깨달음 |
| 검산 순서는? | 행 → 열 → 대각선 순. 대각선은 본 것 같으나 **정확히 기억 안 남. 한쪽만 봤을 수도 있음** |
| "됐다"고 판단한 시점은? | 대각선을 본 **다음** 멈췄으나, **한쪽만** 본 것으로 추정 |
| 깨달은 직후 행동은? | 다른 대각선을 검산해 틀림을 확인하고 숫자를 변경 |
| 숫자 변경 후 확인 범위는? | 행·열·대각선 **전부** 다시 확인 |

## 1. 표면 문제 (잘못된 정의)

솔루션이 섞인 문장 — **검증 전 가설, 사용 금지**

- ❌ "4×4 마방진 **검증 프로그램**을 만든다."
- ❌ "대각선까지 자동으로 체크하는 **솔버**를 만든다."
- ❌ "ECB 패턴으로 **MagicSquare 앱**을 개발한다."
- ❌ "행·열·대각선 합을 한 번에 보여주는 **UI**를 붙인다."

---

## 2. 진짜 문제 (한 문장)

**4×4 부분 마방진에서 빈 칸을 채운 뒤 "맞다"고 판정할 때, 행·열은 확인하지만 대각선 검산을 빠뜨리거나 한쪽만 보고 멈춰서, 같은 판을 20분 넘게 다시 검산·숫자 교체를 반복한 뒤에야 틀림을 알게 된다.**

| 관점 | 내용 |
|------|------|
| **불편** | 같은 판을 반복 검산·숫자 교체 |
| **비용** | 약 20분 |
| **판정** | 한쪽 대각선만 확인하고 "끝"으로 오판 |
| **재현** | "한쪽만 본 거 같아", "정확히 기억이 안 나" |

---

## 3. Mom Test 증거 3줄 (인터뷰 인용)

1. *"빈칸 2개 넣고 행·열·대각선 합 맞췄는데 **대각선 하나를 빼먹어서 20분 날렸어**"*
2. *"**같은 판을 다시 검산**하다가 숫자를 바꿔 넣어 보고 깨달았지"*
3. *"행, 열 순으로 보고 대각선은 본 것 같은데 **정확히 기억이 안 나. 한쪽만 봤을수도 있어**"*

## Mom Test 인터뷰 규칙 (적용 기록)

2. 금지: "만들면 좋겠어?", "이 기능 필요해?", "TDD 하면 어때?"
3. 허용: "마지막으로 ~했을 때", "그때 몇 분 걸렸어?", "뭐 때문에 포기했어?"
4. 모호한 답 → 추궁 1회
5. 솔루션 언급 시 → "그건 나중" 후 불편으로 되돌림
