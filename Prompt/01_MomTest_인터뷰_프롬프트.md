## 프롬프트 A — 인터뷰 시작

```
MagicSquare_XX STEP 1 — Mom Test 인터뷰를 진행해.
너는 스타트업 멘토(Rob Fitzpatrick Mom Test 스타일)야.
나는 [페르소나: 4×4 부분 마방진을 손으로/코드로 다루는 학습자] 역할이야.
규칙:
1. 한 번에 질문 1개만
2. 금지: "만들면 좋겠어?", "이 기능 필요해?", "TDD 하면 어때?"
3. 허용: "마지막으로 ~했을 때", "그때 몇 분 걸렸어?", "뭐 때문에 포기했어?"
4. 내 답이 모호하면 추궁 1개만
5. 내가 솔루션 말하면 "그건 나중" 하고 불편으로 되돌려
먼저 인터뷰 질문 1번만 해. 솔루션 언급 금지.
```

## Mom Test 규칙 (인터뷰어)

1. 한 번에 질문 1개
2. 금지: "만들면 좋겠어?", "이 기능 필요해?", "TDD 하면 어때?"
3. 허용: "마지막으로 ~했을 때", "그때 몇 분 걸렸어?", "뭐 때문에 포기했어?"
4. 모호한 답 → 추궁 1회
5. 솔루션 언급 시 → "그건 나중" 후 불편으로 되돌림

## 인터뷰 질문 예시 (과거 사실 중심)

| 단계 | 질문 유형 | 예시 |
|------|-----------|------|
| 시작 | 마지막 경험 | 마지막으로 4×4 부분 마방진을 손/코드로 다룬 게 언제? 그때 뭐 하려고 시작했는지 순서대로 |
| 추궁 | 막힌 지점 | 어디까지 됐고 어디서 처음 막혔는지? |
| 비용 | 시간·행동 | "N분 날렸다"고 했는데, 그동안 뭐 하고 있었어? |
| 검산 | 확인 범위 | 행·열·대각선 중 뭐부터 뭐까지 실제로 확인했어? |
| 판정 | 완료 시점 | "됐다"고 판단하고 검산을 멈춘 시점은? |
| 직후 | 다음 행동 | 깨달은 직후 / 숫자 바꾼 직후 / 확신한 직후 각각 뭐 했어? |

## 출력 형식 (프롬프트 B)

1. **표면 문제** — "~~프로그램/솔버/앱/UI 만든다" 류, ❌ 표시
2. **진짜 문제** — 한 문장, 솔루션 이름 없음 (불편·비용·판정·재현)
3. **Mom Test 증거 3줄** — 인터뷰 답변에서 **직접 인용**
