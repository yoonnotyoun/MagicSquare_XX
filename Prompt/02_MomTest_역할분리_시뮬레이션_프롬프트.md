# Mom Test 역할 분리 시뮬레이션 — 프롬프트

**프로젝트:** MagicSquare_XX  
**용도:** Mom Test 3턴 가상 인터뷰 생성

---

## 프롬프트

```
역할 분리 Mom Test 시뮬레이션 (MagicSquare_XX).
Role 1 — 인터뷰어: Mom Test 규칙 준수, 질문만
Role 2 — 페르소나: "프로그래밍 수업에서 부분 마방진 과제를 받은 학생"
  (가상이지만 plausible한 과거 사실만)
3턴 진행:
- Turn 1: 인터뷰어 질문 → 페르소나 답
- Turn 2: 추궁 → 답
- Turn 3: 추궁 → 답
마지막: 표면 문제 vs 진짜 문제 (가상 인터뷰임을 명시)
⚠️ 내 실제 경험과 섞지 말 것. 시뮬레이션 라벨 필수.
```

---

## Mom Test 규칙 (인터뷰어 Role 1)

1. 한 번에 질문 1개
2. 금지: "만들면 좋겠어?", "이 기능 필요해?", "TDD 하면 어때?"
3. 허용: "마지막으로 ~했을 때", "그때 몇 분 걸렸어?", "뭐 때문에 포기했어?"
4. 모호한 답 → 추궁 1회
5. 솔루션 언급 시 → "그건 나중" 후 불편으로 되돌림

---

## 출력 형식

- **[시뮬레이션]** 라벨로 시작·종료
- Turn 1~3: Role 1 질문 → Role 2 답
- 마지막: 표면 문제 vs 진짜 문제 표 + Mom Test 증거 3줄
- 가상 인터뷰임을 명시
