# 세션 3 워크북 — 프롬프트

**프로젝트:** MagicSquare_XX  
**용도:** Mom Test 결과 → R-G-I-O + Agent 인프라 워크북 작성  
**관련 보고서:** `Report/03_세션3_워크북_MagicSquare_XX.md`

---

## 프롬프트

```
Mom Test 결과:
- 페르소나: [...]
- 진짜 문제 (한 문장): [...]
- Mom Test 증거 3줄: [...]
MagicSquare_XX 세션 3 워크북을 채워줘:
1) 주제 한 문장 (Mom Test 기반, 솔루션 최소화)
2) R-G-I-O (Role/Goal/Input/Output)
3) 성공 기준 3개 (Mom Test 증거와 연결)
4) 표면 문제 — 이번 프로젝트에서 하지 않을 것
8계층 중 이번 세션에서 만드는 것만: Rule, Command, (Skill), Test Loop
```

---

## 입력 (본 프로젝트 — 02 시뮬레이션 기준)

| 항목 | 내용 |
|------|------|
| **페르소나** | 김서연 (2학년, 프로그래밍 기초 수업) — 4×4 부분 마방진(빈 칸 2개) |
| **진짜 문제** | 행·열만 확인하거나 대각선 한쪽만 보고 "맞다"고 판정 → 15~20분 재검산·숫자 교체 후 틀림 인지 |
| **증거 3줄** | ① 행·열만 확인 ② 한쪽 대각선만 보고 멈춤 ③ 숫자 바꾼 뒤 전부 재계산 |

**근거 문서:** `Report/02_MomTest_역할분리_시뮬레이션_MagicSquare_XX.md`

---

## 출력 형식

1. **주제 한 문장** — 솔루션 이름 없이 불편·비용·오판에 대응
2. **R-G-I-O** — Role / Goal / Input / Output 표
3. **성공 기준 3개** — SC-01~03, Mom Test 증거 1:1 연결
4. **표면 문제** — 이번 프로젝트에서 하지 않을 것 (금지 목록)
5. **8계층 (이번 세션)** — Rule · Command · (Skill) · Test Loop만 상세 기술

---

## 작성 규칙

- 솔루션 먼저 정의 금지 (검증 프로그램·솔버·앱·UI 등)
- 성공 기준은 Mom Test 증거에서 **역추적** 가능해야 함
- Skill은 **(선택)** — WORKBOOK에 워크플로만 적어도 됨
- Test Loop는 **불편 우선** 순서로 ARRR 사이클 정의
- `src/`·`tests/` 구현은 다음 세션(TDD) 범위
